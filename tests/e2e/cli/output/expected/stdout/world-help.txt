
⬡ world - ephemeral runtime instances

Usage:
  spwn world [command]

Lifecycle:
  up                               Spawn a world (see Spawn Flags below)
  ls                               List active worlds
  down <id>                        Destroy a world (agent survives)
  rename <id> <name>               Rename (empty name clears)

Observe:
  inspect <id>                     Inspect a running world's composition and state
  logs <id>                        Show event log for a world
  enter <id>                       Open an interactive shell inside a world

Snapshots:
  snap save <id>                   Save world state
  snap ls                          List snapshots
  snap restore <snap-id>           Rollback to a snapshot
  snap rm <snap-id>                Remove a snapshot

Knowledge:
  knowledge ls <id>                List a world's knowledge files
  knowledge show <id> <path>       Read a knowledge file

Spawn Flags:
  -a, --agent <name>               Agent (repeatable; first is chief)
  -w, --workspace <path>           Host dir to mount (repeatable)
  -c, --config <name>              Named world config
  -n, --name <name>                Display name
  -i, --interactive                Drop into the agent's session after spawn

Examples:
  spwn up --agent neo -w .         Spawn neo in current dir
  spwn up --agent morpheus --agent neo -w .
  spwn ls                          See what's running
